사용 설명서

📝 기록 추가하기

  • 일명: 어떤 일을 했는지 입력합니다
  • 날짜: 그 일을 한 날짜를 선택합니다
  • 주기 (선택): 이상적인 반복 주기를 설정할 수 있습니다
  • 추가 버튼: 클릭하면 기록이 저장됩니다

🔄 정렬 기능

  • 지난일 기준 ▲/▼: 마지막으로 한 일을 기준으로 정렬합니다
  • 남은일자 기준 ▲/▼: 주기가 설정된 항목을 남은 일수로 정렬합니다
  • ▲: 작은 순서대로 정렬 (오름차순)
  • ▼: 큰 순서대로 정렬 (내림차순)

🔧 항목 관리

  • 리프레시: 항목의 날짜를 폼의 기준일로 업데이트합니다 (기본 오늘)
  • 삭제: 항목을 완전히 삭제합니다
  • 히스토리: 클릭하면 이전 기록들을 확인할 수 있습니다

📊 진행도 표시

주기가 설정된 항목은 배경 그라데이션으로 진행도를 시각화합니다.

  • 초록색: 아직 시간이 남음
  • 빨간색: 주기를 초과함 (⚠ 경고)

☁️ 기기간 동기화

좌측 상단 🔐 로그인 버튼으로 Google 계정에 로그인하면 여러 기기에서 자동으로 동기화됩니다. 비로그인 상태에서는 이 브라우저에만 저장됩니다.

  • 충돌 해결: 같은 항목을 여러 기기에서 수정한 경우, 가장 최근 수정이 적용됩니다 (last-write-wins).
  • 오프라인: 인터넷이 끊겨도 정상 동작하며, 온라인 복귀 시 자동 푸시됩니다.
  • 최초 설정: 동기화 기능은 호스팅하는 사람이 Firebase 프로젝트를 연결해야 활성화됩니다. (README 참고)

🔔 푸시 알림

주기에 도달한 활동이 있으면 매일 알려드립니다 (한국시간 오전 9시 기준).

  • 로그인 필요: Google 로그인 후 사용 가능
  • iOS: Safari에서 공유 → 홈 화면에 추가 한 PWA에서만 동작 (iOS 16.4+)
  • 권한: 브라우저가 처음에 알림 권한을 묻습니다. 허용해 주세요.

📅 기준일

입력 폼 상단의 날짜가 모든 기록의 기준일입니다.

  • 추가: 새 항목을 이 날짜로 기록
  • 리프레시: 기존 항목을 이 날짜로 갱신 (기본값은 오늘)
  • 어제·과거 활동을 뒤늦게 기록할 때 유용합니다.
